1. Goal

The goal of this lab was to design and test an IPv4 addressing plan for a small office network using VLSM (Variable Length Subnet Masking). The lab includes subnet calculation, gateway assignment, network segmentation, connectivity testing, and troubleshooting of common IP configuration issues.

2. Scenario

A small company requires a new IPv4 addressing scheme for several departments. The network administrator must divide the available network into multiple subnets with different host capacities using VLSM.

Departments and requirements:

Department	Required Hosts
Administration	50
IT Department	25
Sales		12
Management	6

Available network: 192.168.10.0/24

Tasks:

Design an efficient VLSM addressing plan
Assign subnet masks and default gateways
Configure PCs and routers
Verify connectivity between networks
Troubleshoot possible addressing issues

7. Troubleshooting Scenarios
Scenario 1 – Incorrect Subnet Mask

Problem:
A PC in the Sales department could not communicate with other devices.

Cause:
Incorrect subnet mask configured on the PC.

Solution:
Changed subnet mask from:

255.255.255.0

to:

255.255.255.240
Scenario 2 – Wrong Default Gateway

Problem:
PC could access local devices but not remote subnets.

Cause:
Incorrect default gateway address.

Solution:
Configured correct gateway:

192.168.10.97
Scenario 3 – Interface Shutdown

Problem:
No connectivity between router and switch.

Cause:
Router interface was administratively down.

Solution:

interface g0/1
no shutdown
8. Lessons Learned

I learned how to calculate subnet ranges, apply VLSM for efficient IP allocation, choose proper subnet masks based on host requirements, assign default gateways, and troubleshoot connectivity problems caused by incorrect subnet masks, disabled interfaces, or wrong gateway addresses.

The lab also improved my understanding of network segmentation, router configuration, and IPv4 connectivity verification using Cisco IOS commands.
